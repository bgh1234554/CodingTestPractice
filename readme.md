## 학습 방법 📝
1. 코드를 보고 로직과 코딩 방법에 대한 구상한다.
2. 최대 30분 동안 문제의 로직을 생각하고 구현해보는 시간을 갖는다.
3. 만약 30분 이상 고민해도 안되는 문제들은 풀이를 보고 다시 한번 생각을 정리한 다음에 풀이를 진행한다.
4. 코드로 구현을 못하겠으면, 적어도 의사 코드(Pseudocode)라도 작성해서 후에 해답 코드랑 비교한 뒤 구현한다.
5. 코딩이 끝난 이후에는 코드를 보며 풀이 과정과 핵심 아이디어를 요약해서 주석에 적는다.
6. 왜 풀이를 떠올리지 못했는지 이유를 적으면서 복기한다.

### [Neetcode 사용 가이드](https://neetcode.io/courses/lessons/how-to-use-neetcode-effectively)에서 추천하는 공부 방법 📘
#### 공부 원칙 📝
1. 어떻게든 꾸준히 매일 한문제씩이라도 푼다.
2. 절대 해답 코드를 복사/붙여넣기 하지 않는다.
3. 절대 몇시간동안 코딩 문제를 붙들고 있지 마라. 모든 코테 문제는 처음에 수학자들이 푼 것이다. 쉽게 풀릴리 없다.
4. 이해 없이 무조건 외우려고만 하지 마라

#### 문제 풀이 방법 📝
1. 처음에는 혼자서 풀이를 시작하라.
2. 종이에 손으로 푸는 것도 시도해봐라.
3. 15~20분동안 전혀 손을 대지 못하겠으면 풀이를 봐라.
4. 만약 한시간동안 풀이를 봐도 이해하지 못하겠으면, 더 쉬운 문제로 넘어간다.
5. 이전에 어려웠던 문제를 다시 풀어본다. 이 때는 풀이를 보지 않고 시도한다.

    * Neetcode 150은 필수 문제만 모아놓은 것이며, 초보자에게 더 친절한 버전인 Neetcode 250과,

## 백준과 리트코드 난이도 비교 (GitHub Copilot의 설명) 📊

백준의 문제 난이도는 다음과 같이 나뉜다:

1. **브론즈 (Bronze)**: 기초적인 문제들로, 프로그래밍을 처음 배우는 사람들에게 적합하다.
2. **실버 (Silver)**: 기본적인 알고리즘과 자료구조를 이해하고 있는 사람들에게 적합하다.
3. **골드 (Gold)**: 중급 알고리즘 문제로, 다양한 알고리즘과 자료구조를 깊이 있게 이해해야 한다.
4. **플래티넘 (Platinum)**: 고급 알고리즘 문제로, 매우 복잡한 알고리즘과 자료구조를 다룬다.
5. **다이아몬드 (Diamond)**: 매우 어려운 문제들로, 알고리즘 대회에서 상위권을 목표로 하는 사람들에게 적합하다.
6. **루비 (Ruby)**: 최상위 난이도의 문제들로, 알고리즘 대회에서 우승을 목표로 하는 사람들에게 적합하다.

리트코드의 난이도와 비교하면 다음과 같습니다:

- **리트코드 Easy**: 백준의 브론즈 ~ 실버 하위
- **리트코드 Medium**: 백준의 실버 상위 ~ 골드 중위
- **리트코드 Hard**: 백준의 골드 상위 ~ 플래티넘, 다이아몬드, 루비

이 비교는 대략적인 기준이며, 문제의 성격에 따라 다를 수 있다.

### 실제 코딩 테스트 난이도? (ChatGPT의 설명) 🧩
대부분의 기업은 LeetCode Medium ~ Hard 초반 수준의 문제를 출제한다.
#### 해외 대기업 (Big Tech & FAANG 수준)

*    난이도: LeetCode Medium 2~3문제 + Hard 1문제
*    주제: 그래프 탐색, DP, 이분 탐색, 세그먼트 트리, 트라이, 힙/우선순위 큐
*    예: Google, Meta, Amazon
*    DFS/BFS, DP 응용 문제
*    Union-Find, 트라이(Trie) 활용 문제
*    비트마스킹 & 백트래킹

#### 국내 IT 대기업 (네이버, 카카오, 삼성 SW 역량테스트)

*    난이도: LeetCode Medium 2~3문제
*    주제: 문자열 처리, 그래프 탐색, DP, 최적화 알고리즘
*    예:
*    삼성 SW 역량테스트: 구현, 시뮬레이션, 백트래킹
*    카카오 코딩테스트: 문자열 처리, BFS/DFS, 트리
*    네이버/라인: 그리디, DP, 최단 경로

#### 일반적인 IT 기업 중 코딩 테스트를 보는 기업

*    난이도: LeetCode Easy 1~2문제 + Medium 1문제
*    주제: 배열 조작, 정렬, 해시맵, 스택/큐, 기본적인 DFS/BFS
*    예: 중견·스타트업 (당근마켓, 쿠팡, 배달의민족 등)
*    해시맵을 활용한 문제
*    간단한 BFS/DFS (미로 탐색 등)

#### 결론
* 국내 IT 대기업 & FAANG → LeetCode Medium 2~3 + Hard 1
* 국내 IT 대기업 → LeetCode Medium 2~3
* 국내 중견 IT 기업 → LeetCode Medium 1~2
* 일반 IT 기업 → LeetCode Easy 1~2 + Medium 1

즉, 백준 실버 후반\~골드 초반이 일반적인 기업 코딩 테스트 대비 기준이고,
골드 후반\~플래티넘 초반을 풀 수 있으면 상위권 기업 대비도 가능하다.
